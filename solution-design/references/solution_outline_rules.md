# Solution Outline Rules

The outline freezes the proposal framework before formal drafting.

## Required Content

Include:

- title and project orientation
- selected technical route
- chapter structure
- estimated page allocation
- technical architecture
- module decomposition
- module inputs and outputs
- indicator system
- verification logic
- figure placeholders with figure number, title, and note
- user-confirmation questions

## Page-Count Guidance

Recommend page counts by complexity:

- 5-8 pages: single-route, low-complexity proposal
- 8-15 pages: ordinary algorithm or engineering proposal
- 15-25 pages: multi-module research or engineering design
- 25+ pages: complex system proposal with multiple subsystems, datasets, or validation layers

## Outline Quality

Do not use a fixed universal chapter template. Derive chapters from project goals, route logic, input/output structure, module boundaries, indicators, and verification needs.

Do not include implementation code, simulation scripts, test plans, or fabricated results.
