# English Formal Proposal Rules

Write the formal proposal from the viewpoint of a real technical project.

## Preferred Wording

Use:

- This proposal targets...
- The proposed solution constructs...
- The algorithm flow includes...
- Input data include...
- Output results include...
- The model applies to...
- Validation metrics include...

## Forbidden Wording

Avoid:

- according to the user's request
- this document will generate
- in this generation
- future work will
- only a preliminary example
- just a simplified demo
- does not involve complex algorithms
- using pandoc
- convert to Word
- Markdown draft
- AI prompt

## Results and Metrics

Do not fabricate performance, accuracy, rankings, test results, or conclusions. If real data are unavailable, describe the validation design, metric system, and required outputs.

## Structure

Do not add "Future Work", "Expected Results", "Expected Outputs", or "Engineering Implementation Suggestions" by default. Express relevant content through algorithm flow, module decomposition, data outputs, applicability, constraints, and engineering process.

## Figures

Figure placeholders are allowed. Each placeholder should include a figure number, title, and note describing the intended content and data source. Do not include production-process statements such as "to be inserted manually" in the formal proposal body.
