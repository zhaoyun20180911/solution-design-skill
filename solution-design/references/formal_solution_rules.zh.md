# 中文正式方案写作规则

正式方案必须采用真实项目方案视角撰写。

## 推荐表达

可以写：

- 本方案面向……
- 本方案构建……
- 算法流程包括……
- 输入数据包括……
- 输出结果包括……
- 该模型适用于……
- 验证指标包括……

## 禁用表达

不得写：

- 根据用户要求……
- 下面将生成……
- 本次先……
- 后续可以……
- 这里只是……
- 为了演示……
- 本方案只是简化演示
- 本方案不涉及复杂算法
- 使用 pandoc 转换为 Word
- Markdown 初稿
- AI 提示词

## 结果与指标

没有真实数据时，不得编造效果、精度、排名、测试结果或结论。应写验证设计、指标体系和数据输出要求。

推荐写法：

```text
方案输出包括误差序列、统计指标和过程曲线，用于支撑算法性能评价。
```

## 章节结构

除非用户要求，不默认设置“后续工作”“预期效果”“预期输出”“后续展望”“工程实现建议”等章节。将相关内容写入算法流程、模块划分、数据输出、适用条件、约束条件和工程处理流程。

## 插图

可以预留图位，但第一版不实际绘图。图位应自然出现在模型、流程、架构或验证说明附近，包含图号、图题和备注。不要把“后续手动插入”“本次不绘制”等制作过程写入正式方案正文。
