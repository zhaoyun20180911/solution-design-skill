# Research Pack Rules

The research pack supports route selection. It is not the final proposal body.

## Required Sections

1. Project question and research scope
2. Route candidates
3. Algorithm and method notes
4. Standards, technical documents, papers, and open-source references
5. Learning-resource list for the user
6. Comparison table
7. Recommended route and reasons
8. Open questions for user confirmation

## Source Integrity

Never invent bibliographic metadata, DOI values, standards, links, datasets, benchmark results, or tool capabilities. If live research is unavailable, provide search keywords, databases to search, and a list of source types the user should provide.

Use source-grounded wording:

- "The available source indicates..."
- "The document supports..."
- "This route should be verified against..."

Avoid unsupported wording:

- "The literature proves..." without sources
- "The method reaches..." without data
- "Existing evidence shows..." without verifiable evidence

## Route Comparison Dimensions

Compare candidate routes by:

- function and role in the project
- applicable conditions
- input data and assumptions
- output form
- implementation difficulty
- verification difficulty
- engineering integration risk
- required expertise or learning materials
